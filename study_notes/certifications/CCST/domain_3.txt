	• Some of the key roles of operating systems:
		○ Identification
		○ Authentication
		○ Authorisation 
	○ The first interaction with any operating system is usually the login screen 
			§ Where subject provides identity, and operating system challenges identity
				□ Requests authentication credentials
			§ After the identity has been authenticated, the authorisation layer takes control and decides what that identity is authorised to do
			§ As well as providing authentication and authorisation, operating systems can provide additional controls that limit access to connect resources.

	• Host Based Firewall
		○ Operating systems manage interaction with hardware resources
			§ Networking hardware falls in that domain
			§ The operating system is a good place to place controls over network traffic
				□ The operating system is well-positioned to enforce rules over what traffic is authorised and what is not
			§ Most operating systems include firewall capabilities
	• Command Line Interface (CLI) - Typing individual commands to interact with the operating system
		○ Still a common interface
		○ Less popular because it is harder to use then a GUI
			§ Requires users to remember many command names and parameter formats
		○ Lends itself well to repetition and automation
		○ Most CLIs provide the ability to chain multiple commands together
			§ The output of one command can be routed to the input of another command.
			§ Simple commands can be used to construct very complex actions
		○ Cybersecurity professionals and attackers rely on the power of the CLI to carry out many security related functions
			§ For a while, the only available CLI in windows was the command prompt
			§ Then Windows introduced Windows PowerShell, a more powerful CLI
			§ Supports many internal commands to interact with the operating system functionality
				□ Implements advances scripting capabilities
				□ Today, PowerShell is the first choice for administrators and security professionals
				□ PowerShell has become so popular that Microsoft has released the CLI as an open source project for a variety of operating systems, so today PowerShell runs on most current operating systems, not just Windows
				□ PowerShell gives the ability to access many built in commands as wells as external programs
					® (eg: dir shows a list of folders
		○ Support executing commands stored in text files
		○ The text files are very simple programs with lists of commands and simple flow control
			§ They can responds to changing environmental conditions
	• Scripts - Text files that contain commands
		○ Make it possible to construct highly complex processes that are easy to initiate
	• File and Directory Permissions - Building an access matrix
		○ Each file or directory to be secured needs a defined list of accounts or groups and what each account or group is able to do
		○ The account or group list, along with a row containing the permissions for that identity, can be expressed as a table or matrix.
		○ When a subject requests a specific type of access to a protected resource or object, the operating system searches the access table to find the row that corresponds to the affected account or group
		○ Based on entries in that row, the operating system either grants or denies the access request.
		○ Matrix is often referred to as an access control list (ACL)
	• Linux and macOS handle file and directory permissions differently
			§ Each file or directory have unique permission lists
				□ Instead of creating ACLs
				□ Permissions are based on three groups of identities:
					® File owner
					® Any other identities belonging to the group that is associated to the file
					® Anyone else
				□ When a subject requests access to a file under protection, the OS determines if the user ID is the owner of the file, if they are associated in a group with the owner, or if they are neither
					® Access is either granted or denied depending on the permission settings for that resource and the classification of the subject 
					
	• Some of the primary functions of the operating system are AAA
		○ The main purpose of AAA is to enforce access controls
	• Access Control - Any control that limits access to protected resources based on an identity and its authority
			§ Role based access control
				□ Job roles are commonly implemented as user groups in Windows, Linux and macOS
		○ Proved much of the foundation for informational security
		○ The most common class of access controls are implemented to restrict access to files and directions
		○ Most operating systems nowadays use permissions to enforce access controls
	• Permissions - Definitions of actions that are allowed or denied based on an identity association
	• Identity - Can refer to an account or security group to which an account belongs
	• Privilege Escalation
		○ When a user logs into a system, the authorisation layer is responsible for determining what that identity is able to do
			§ Attackers will usually start will limited privileges, and then attempt to escalate those privileges
				□ Attackers will do this by:
					® Spoofing identities
					® Engaging in social engineering
					® Compromising access controls and adding unauthorised privileges and permissions to their sessions 
			§ One of the jobs of a cyber professionals is to stop privilege escalations
				□ Prompted to elevate permissions for a short period of time
				□ Some functions require elevated permissions to run for a limited time to carry out certain activities
			§ A program may run with elevated permissions to carry out an authorised function on behalf of a limited permission subject
			§ Attackers sometimes attempt to generate an error or break out of a program during an elevated privileges sections
				□ If an attacker can break out of an elevated permissions session, the attacker may end up at a command line while retaining elevated permissions.
				□ This is a dangerous outcome and can lead to dangerous attacks.
- Endpoint Tools
	• Netstat Command - Held display what connections a particular node uses to communicate with other nodes
			§ Most operating systems include the netstat command, typical as a CLI utility
			§ Usage and utput are slightly different for different operating systems 
		○ Primary use for netstat is to determine what ports are in use for network communication.
	• Nslookup - A CLI utility that helps in troubleshooting DNS servers
			§ Takes a hostname as input and return the IP address associated with that hostname
		○ An IP address that is input will return the hostname it is associate with
		○ Can be used to determine if your DNS server is working properly, if it has been compromised, or if it is returning unexpected information.
		○ Uses the primary DNS server from your IP settings by default
		○ Can be instructed to use a different DNS server
		○ Can be used to update DNS server settings
	• Tcpdump
			§ Every good cybersecurity utility toolbox should include the tcpdump utility
		○ Allows the user to monitor TCP network traffic and displays the contents of the packets received from or sent to connect network
		○ The tcpdump utility provides many options that allow network traffic to be filtered so that the output only contains what you want to see
			§ Traffic can also be seen on specific interfaces or subnetworks
		○ Makes it easy to view what activity is happening on your network
- Network Scanners
		○ The best approach to maintaining awareness of network assets is to frequently scan your networks and compare the results to a hardware inventory list
			§ Differences should be noted and investigated
		○ The primary reason for having network asset lists is to ensure that all nodes are authorised and complying with network connection requirements
			§ Any insecure node risks the security of the entire network
			§ A network is only as secure as its weakest/least secure node
		○ As well as knowing what nodes exist on a network, its important to document the configuration of each node with respect to network connection parameters
		○ It is important to manage the services on all nodes and control how they are accessed internally and externally 
			§ This level of control is not possible unless there is a clear understanding of what assets are connected to a network.
	• Nmap Utility
		○ Useful in identifying nodes based on network activity
- Software Inventory and Program Deployment
	• Every network node is a computing device 
	• Computing devices are comprised of hardware & software
			§ Hardware - The physical aspect of any device or computer
			§ Software - Provides the instructions for how the device operates
			§ Firmware - Software that has been installed on a chip and now cannot be changed without the removal of said chip
	• Hardware nodes have no functionality without software
		○ Software provides the utility value of the node and offers potential vulnerabilities to attack
- Data Backups
	• Several types of integrity and availability attacks (such as ddos) result in data that is either untrusted or unavailable
		○ The best way to respond to such attacks is to restore data to a trusted and available state
		○ Key to a good backup strategy is to:
				□ Implementing these goals can be difficult
			§ Create frequent copies
			§ Store them in secure locations
			§ Have sufficient procedures in place to restore them
		○ Creating a backup policy includes identifying what data is being backed up, how it is being backed up, how many back ups are required, and the method to transport and store backups in their final locations
			§ What, where, when, how
		○ Backups are useless unless there is a well planned restoration process that is prepared to restore them to their original data locations
		○ Multiple local backups in secondary locations combined with multiple cloud backups can result in multiple layers of resilience
			§ The goal is to restore operations after an interruption as soon as possible
			§ Backups are good, but only if the restoration process is good as well
- Regulatory Compliance
	• Compliance Requirements
		○ Both a curse and a blessing
		○ Although it may seem that organisations take unusual or unnecessary action to comply with requirements, those organisations will often become more secure in the process
			§ Compliance is not security, but the minimum level of security
		○ Several compliance requirements place restrictions and performance madidates on organisations
			§ The specific performance mandates under which your organisation must comply depends on the nature of your organisation.
		○ Important compliance requirements:
			§ PCI DSS - Payment Card Industry Data Security Standard
				□ Applies to any organisations that accepts, processes or stores payment data
			§ HIPAA - Health Insurance Portability and Accountability Act
				□ Applies to any health care organisation that collects, processes or stores personal healthcare information
			§ FISMA - Federal Information Security Management Act
				□ Applies to a range of organisations that collects, processes or stores information regarding the US government
			§ FERPA - Family Educational Rights and Privacy Act
				□ Governs access to educational information, specifically data related to students
			§ GDPR - Genera Data Protection Regulation
				□ Applies to any organisation that interacts with European union citizens
	• BYOD - Bring your own device
			§ When subjects use their personal devices
			§ Is becoming a growing target for security attacks
		○ BYOB policies establish rules and regulations of how personal devices are to be used, to maintain security
		○ Organisations set BYOB guardrails by establishing policies and implementing NAC
			§ Policies and NAC implementation can define requirements for how devices are managed, whether data stored on those devices is encrypted, how apps are authorized for installation on personal devices, and what configuration requirements are places on such devices before access is granted
		○ BYOB policies ensure that personal devices do not pose a risk to the organisation.
- Windows and Application Updates
	• As security researchers find new vulnerabilities and new attacks, they are reported to hardware and software vendors and publicly available vulnerability repositories
		○ Both hardware and software vendors pay attention to vulnerabilities that affect their products and generally do a god job of providing updates to address discovered vulnerabilities.
	• To take advantage of proactive vendor actions it is important to identify and then apply any updates that your hardware of software vendors have released.
		○ An aggressive security posture must include frequent operating system and application updates
	• All major operating systems include:
		○ Integrated capability to check for available updates of operating system components
			§ Download those updates
			§ Apply them to running systems
	• Most software applications include a similar capability to check for updates and then download and apply updates that include software to address newly discovered security vulnerabilities.
- Device Drivers
		○ All devices such as printers, keyboards, gaming controls and other hardware components interact with an operating system through device drivers that handles the technical details of communication with and operating system.
	• Software programs
	• May need updates as new features are added and software vulnerabilities are discovered and patched
		○ Be aware of new device drivers released by the hardware vendors that your company utilises to ensure optimal and secure connection
		○ Acquire and apply the latest version of the device driver
	• Many hardware devices have native software that is burned or flashed into an onboard chip to control basic operation
- Firmware - Software that is transferred to an onboard chip
	• Firmware updates are useful for the same reason as software updates
		○ They include functionality enhancements and changes for discovered security vulnerabilities
	• Firmware changes can often be more difficult then software
		○ The process involves downloading an updated firmware image and then executing a process to transfer the software to the onboard chip
	• Maintaining a detailed inventory of assets can help track out-of-date firmware
- It is important to use the latest security updates available for all hardware and software components in your environment
- Patching - Applying updates
	• Patching firmware and software can dramatically reduce or eliminate known security vulnerabilities
	• One of the most active tools available to security professionals for managing the security of an environment
- Event Viewer - The primary tool for monitoring Microsoft environments
		○ An important tool in managing security for any environment is monitoring its performance and reacting to any adverse reports
	• Hardware components, device drivers, the operating system, and application all post notable offenses to the Event Viewer logs
	• Provides a convenient interface for the collection of logs
		○ The logs provide visibility into the operation of the Windows system
	• Understanding Event Viewer can help security professionals determine whether issues exist in a Windows environment and what to do about them if they do
- Audit & Log files
		○ Nobody expects software applications to work perfectly
	• One of the main problems of software operation is conveying errors, warnings, and any messages of interest to the appropriate recipient.
		○ The most common technique used for applications to convey important information is log files
	• Operating systems, applications, and any other software components commonly report different levels of messages in their log files
		○ Every operating system has its own log file strategy, and many application software packages do as well
	• It is important for cybersecurity professionals to be familiar with their target operating system mechanisms for reporting messages
		○ While the Windows operating system uses the Event Viewer, Linux and macOS operating systems use a different approach based on collection audits and logging messages from text files
	• Even within each operating system, many applications create their own log files with unique internal formats to report notable events
		○ In addition to understanding the logging structure of operating systems, cybersecurity professionals should have a good understanding of the log files produced by the applications their environments use
	• Log files are valuable to security professionals and attackers alike
	• System and application log files contain audit trails for both expected and unexpected activity
	• Many attacker activities are recorded in various log files
	• Cybersecurity professionals and investigators commonly examine log files
		○ This is to uncover attacker techniques and identities
		○ Attackers frequently attempt to remove incriminating log file activity to maintain anonymity
			§ If an attacker can remove evidence of their activity, they dramatically reduce the possibility of being caught
- Syslog - Separates log file message generation from log file message storage // makes if difficult for attackers to cleanse log lifes from messages that record their activities
	• It is recommended to implement syslog for environments that depend on the integrity of log files to carry out detection and post-incident investigation
	• Cybersecurity professionals implement strategies to protect log file entities once they are reported
		○ To avoid he possibility of log file tampering 
		○ One approach to protecting log files is to transfer the log file entries to a secure remote location where they would be difficult to alter
			§ The Linux syslog utility allows systems to transmit logging messages that are resistant to modification
- Anomaly Identification
	• System and application log files can provide valuable input to malicious activity detection and post-incident investigations
		○ The value of any log file entirely depends on the ability to separate normal and abnormal behaviour 
			§ Log file entries merely report actions that have occurred in the past
		○ It is up to the analyst to determine whether a reported activity represents normal behaviour or potentially malicious behaviour
			§ Although log file analysis can be automated, it still requires understanding and recognition of normal behaviour
	• The best way to identify malicious behaviour is to understand what normal behaviour looks like
	• A productive activity of cybersecurity professionals is to examine normal behaviour in their environment on an ongoing basis
		○ Doing so makes it easier to detect unusual and potentially malicious behaviour
- Scanning Systems and Logs
	• Malicious software is one of attackers' favourite techniques to either initiate or continue ongoing attacks
		○ There is no silver bullet when it comes to malware protection, but a comprehensive malware control layer can help in reducing the risk of infection
		○ Antimalware software comes in two forms:
			§ The first and most common approach is to scan files and network messages and compare their contents to a database of known malware signatures
				□ Any time a match is detected the software issues an alert to indicate that further investigation is necessary 
			§ The other approach is using heuristics
				□ Instead of comparing individual blocks of data against a database of known malware signatures, the nature of the behaviour of processes and traffic handling services is examined.
					® The idea is that malware behaviour is distinct and detectable
- Malware Remediation
	• There is always a possibility that malware will make it into your environment
		○ When malware is detected and cannot be remediated automatically it is important to follow well-thought-out malware remediation
	• The first step is to contain the damage and infection by removing any detected infected systems from the network
	• Do not forget other connection opportunities such as Bluetooth, infrared, or any other wireless media a device supports
		○ Once an infected device is disconnected from the network, the next step is to explore options from removing the malware from the systems
	• Malware removal may not be feasible in some cases and may require wiping and reinstalling all software components on a device or computer
	• A good strategy is to assume that malware or some other attack could incapacitate any device at any point in time
		○ Pessimistic planning tends to lead toward remediation strategies that work should an attack completely compromise any device on your network.
- Good planning always results in better and more timely outcomes
