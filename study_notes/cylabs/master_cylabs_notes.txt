ARTIFICIAL INTELLIGENCE

	CyLab AI Foundations I

		Perception - defn: a computational model inspired by how neurons in biological brains work - foundation of neural networks
			- Like a binary decision maker (a neuron that fires or doesn't based on input)
			- 1D perception takes 1 input, 2D perception takes two, and so on... a decision boundary is the turning point where inputs result in the perception firing or not firing.
				A line boundary is like a line that seperates the 2D cartisian plane where a co-ord represents a combination of inputs, where co-ords/input combos on one side result 
				in the perception firing, and those on the other side result in it not firing. i.e. a 2D decision boundary
			- A plane would do the same thing in a 3D perception, and the term 'hyperplane' is used to describe boundaries in higher dimensions
			- Example of the mathematical logic that perceptions use

			output = {1 if w.x+b >= 0} {0 otherwise}
	
				x = input (or inputs if multidimensional)
							
				w = weight - how much each input matters	

				b = bias - shifts the boundary position

				w.x+b -> weighted sum plus the bias

			- Classification - defn: sorting things into categories (binary classification has two categories)
				- Perceptions are perfect for binary classifications, but the right weight and bias must be found
					- This can be done manually for smaller data sets
					- Training is required for large data sets
						defn: automatically finding the right weights and biases based on examples in the data
						
				 		Perception Learning Algorithm (how training works, conceptually)
				 			1. Begin with random weight
				 			2. Make predections and measure how many errors there are
				 			3. Adjust the weights* slightly in a way that reduces errors
				 			4. Repeat steps 2 and 3 until the errors stop decreasing (or reach zero)

				 			NOTE: Perceptions can only solve linearly seperable problems (where a straight line, or hyperplane, can seperate the categories)

				 				Famous example: XOR Problem

				 					takes two binary digits (0 or 1) - equates to 0 if they are the same, equates to 1 if they are different

				 				This limitation inspired the developement of neural networks - multiple perceptions combined together with the ability to solve non-linearly-seperable problems

				 		* when adjusting the weights, how much you adjust them is called the Learning Rate (measured in steps)
				 				- Learning rate is a hyperparameter: it's set before training begins
				 			- Smaller learning rate (e.g. 0.01) is slow but stable
				 			- Medium learning rate (e.g 0.1 - 0.5) is balanced
				 			- Large learning rate (e.g. 1.0) is agressive and risks overshooting and oscilating

		AI Modalities: 
			- Assistents: primarly just answers questions and responds; they won't act unless a human tells them too
			- Agents: take actions autonomously using tools and memory
				- Agent actions can have input, so guardrails (e.g. human approval) make autonomy safer

		Key Ethical Principles:			
			
			Transparency: People should understand how and why an AI system made a decision about them.

			Accountability: Someone should be responsible when an AI system makes mistakes or causes harm.
			
			Fairness: AI systems should treat different groups equitably and not perpetuate discrimination.
			
			Verification: You should verify that an AI system actually does what you think it does. Trust is good; trust but verify is better.
			
			Understanding Limitations: Knowing what an AI system cannot do (like a perceptron failing on non-linearly-separable problems) is crucial for using it responsibly.
